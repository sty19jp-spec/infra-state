# Incident: ASR502 (2026-01-20)

## 症状
- https://asr.neurobase-lab.com/health → 502 Bad Gateway

## 原因
- NPM upstream (172.17.0.1:8000) に待受プロセスが存在しなかった
- ASRは /opt/asr の FastAPI アプリだが未起動だった

## 対応
- /opt/asr/asr_server.py を uvicorn で起動
- 起動方式を systemd に統一
- asr.service を作成し enable

## 現在の正
- ASR: systemd 管理
- Listen: 0.0.0.0:8000
- NPM upstream: 172.17.0.1:8000
- health: HTTP/2 200
