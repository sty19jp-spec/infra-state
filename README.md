# infra-state

## 目的
本リポジトリは、XserverVPS 上で運用している以下サービスの
**構成情報・運用手順・復旧手順を一元管理**するための台帳です。

- n8n
- Dify
- ASR
- Nginx Proxy Manager

## 重要な参照先（正本）
- **context.md** を最初に参照してください  
  - Google Drive 上の正本フォルダ
  - 稼働URL
  - 最新スナップショット

👉 このリポジトリ内の情報は  
**「再現・判断に必要な最小情報」**のみを扱います。

## 置くもの / 置かないもの
### 置くもの
- `.md`（運用・判断・復旧の文章）
- `.yml`（再現可能な構成）

### 置かないもの
- `.env` / APIキー / パスワード
- 実行ログや一時ファイル
- スクリーンショット（Driveで管理）

## 運用ルール（最低限）
- 変更前後で **context.md の最終更新日**を更新する
- 迷ったら **Driveの正本 → Git → 実機** の順で確認する

## このリポジトリについて 2026/01/20

本リポジトリは、XserverVPS 上で運用している以下サービスの  
**構成情報・運用手順・障害対応手順（Runbook）を一元管理**するためのものです。

対象サービス:
- n8n
- Dify
- ASR (Speech-to-Text)
- Nginx Proxy Manager

## 重要な方針

- 本リポジトリには **秘密情報は一切含めません**
  - APIキー
  - パスワード
  - トークン
  - 秘密鍵
- 生ログ・バックアップ・export データは GitHub には置きません
  - Google Drive 側で管理します（詳細は context.md）

## 運用ルール（必須）

- 変更前 / 変更後で必ず Snapshot を取得します
- Runbook は runbooks/ 配下を正とします
- 実作業は CUI（SSH / PowerShell）前提です

## 想定読者

- 自分（将来の自分）
- 同様の構成を検討している技術者
- 手順・考え方の参考用途

※ このリポジトリは「再現可能な手順」と「思考の記録」を目的としています。
